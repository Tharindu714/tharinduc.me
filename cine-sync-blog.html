Code Highlight: Concurrent Alert System

One of the challenges I faced was developing a concurrent alert system that sends notifications every 15 minutes. The following snippet demonstrates the core logic behind scheduling these alerts, though background execution remains a work in progress.

Android Alert Scheduling (Java-code)

AlarmManager alarmManager = (AlarmManager) getSystemService(Context.ALARM_SERVICE);
Intent intent = new Intent(this, AlertReceiver.class);
PendingIntent pendingIntent = PendingIntent.getBroadcast(this, 0, intent, 0);

// Schedule alert every 15 minutes
alarmManager.setRepeating(AlarmManager.RTC_WAKEUP, System.currentTimeMillis(), 
    15 * 60 * 1000, pendingIntent);

Note: The alert system occasionally faces issues with background execution due to OS battery optimizations. I’m actively working on a robust solution to ensure consistent performance.

Code Highlight: Sorting & Filtering Movies

Below is a valuable code snippet that demonstrates how to sort a list of movies alphabetically by their names and then filter that list based on a search query. This can be particularly useful for creating a searchable, alphabetically ordered movie database in your CineSync app.

Adding movie filters (Java-code)

            import java.util.ArrayList;
            import java.util.Collections;
            import java.util.Comparator;
            import java.util.List;
            import java.util.stream.Collectors;
            
            // Assume a simple Movie class with a 'name' property
            public class Movie {
                private String name;
                // ... other properties
            
                public Movie(String name) {
                    this.name = name;
                }
            
                public String getName() {
                    return name;
                }
            
                // Optional: Override toString() for debugging
                @Override
                public String toString() {
                    return name;
                }
            }
            
            public class MovieFilter {
                public static void main(String[] args) {
                    // Sample movie list
                    List<Movie> movies = new ArrayList<>();
                    movies.add(new Movie("Inception"));
                    movies.add(new Movie("Avatar"));
                    movies.add(new Movie("The Matrix"));
                    movies.add(new Movie("Interstellar"));
                    movies.add(new Movie("Avengers: Endgame"));
            
                    // 1. Sort the movies alphabetically (case-insensitive)
                    Collections.sort(movies, new Comparator<Movie>() {
                        @Override
                        public int compare(Movie m1, Movie m2) {
                            return m1.getName().compareToIgnoreCase(m2.getName());
                        }
                    });
            
                    System.out.println("Sorted Movies:");
                    movies.forEach(System.out::println);
            
                    // 2. Filter movies by a search query (e.g., "av")
                    String query = "av";
                    List<Movie> filteredMovies = movies.stream()
                        .filter(movie -> movie.getName().toLowerCase().contains(query.toLowerCase()))
                        .collect(Collectors.toList());
            
                    System.out.println("\nFiltered Movies (query: \"" + query + "\"):");
                    filteredMovies.forEach(System.out::println);
                }
            }
Sorting:

  • The code uses 
    Collections.sort()
    with a custom Comparator to compare movie names without considering letter case.
  • This ensures the list of movies is arranged alphabetically.

    • Filtering: 

    filter()
  • This method checks if each movie's name contains the given search query (converted to lowercase for case-insensitive matching).
  • The results are collected into a new list 
    (filteredMovies)

    • Usage in Your App

  • Integration: Use similar logic in your Android project when you fetch movie data from your database. You can sort the list once the data is loaded and then filter it based on user input from a search bar.
  • Real-Time Search: Combine this with an EditText listener to update the filtered list as the user types, ensuring a dynamic and responsive search experience.

    • Challenges & Ongoing Improvements

    Concurrent Alert System

    Implementing a concurrent alert system that triggers every 15 minutes proved challenging. Background duties on mobile devices can interfere with scheduled tasks, leading to inconsistent notifications. While the basic logic works, intermittent issues still occur during background operation. I’m currently exploring alternative solutions, such as using WorkManager for more reliable background processing.

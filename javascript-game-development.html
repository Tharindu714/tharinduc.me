Why HTML5 Canvas & JavaScript?

The following technologies were used to build this interactive game:

  • HTML5 Canvas: Provides a powerful 2D rendering context for dynamic graphics and animations.
  • JavaScript: Enables event handling, game logic, and DOM manipulation, making the gameplay interactive and engaging.
  • Vanilla CSS: Used for styling the game interface and ensuring a consistent user experience.

JavaScript (Game Event Handling & Canvas Rendering)

// Set up the canvas and context
const canvas = document.getElementById('gameCanvas');
const ctx = canvas.getContext('2d');

// Game variables
let cowboy = { x: 50, y: canvas.height - 100, width: 50, height: 50 };
let obstacles = [];
let score = 0;

// Listen for keyboard events
document.addEventListener('keydown', function(event) {
  if (event.code === 'ArrowUp') {
    cowboy.y -= 20;
  }
  if (event.code === 'ArrowDown') {
    cowboy.y += 20;
  }
});

// Main game loop
function gameLoop() {
  ctx.clearRect(0, 0, canvas.width, canvas.height);
  // Draw cowboy
  ctx.fillStyle = 'brown';
  ctx.fillRect(cowboy.x, cowboy.y, cowboy.width, cowboy.height);
  // Update and draw obstacles...
  // Score tracking logic
  score++;
  requestAnimationFrame(gameLoop);
}
gameLoop();

Challenges Encountered & How I Overcame Them

1. Ensuring Smooth Event Handling

Initially, managing rapid key presses caused lag in the game. I refined the event listeners and implemented requestAnimationFrame to optimize performance and create a smoother gameplay experience.

2. Balancing Game Difficulty

Adjusting the obstacle frequency and speed was challenging. Iterative testing helped fine-tune the balance between challenge and fun, ensuring the game remained engaging while being accessible for beginners.

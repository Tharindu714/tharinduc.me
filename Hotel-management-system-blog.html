Tranquil Terrace HMS is a large-scale hotel management solution built using Java and MySQL. Developed as a group project, this system manages reservations, guest check-ins/outs, billing, and employee schedules across multiple departments, such as Administration, HR, Front Office, and Kitchen.

Why Java & MySQL?

For Tranquil Terrace HMS, the decision to use Java and MySQL was driven by the need for a robust, scalable, and secure system. Here’s why:

  • Java: Offers platform independence, robust performance, and a vast ecosystem of libraries—ideal for developing the core business logic and managing complex processes in a hotel environment.
  • MySQL: Provides reliable, efficient, and scalable relational database management, making it perfect for handling all transactional data, such as reservations, billing, and employee records.

Code Highlight: JDBC Database Connection

One of the critical aspects of the system is establishing a secure connection to the MySQL database using JDBC. Below is a sample code snippet for connecting to the database:

Java Code: Connecting to MySQL with JDBC

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class DatabaseConnection {
    private static final String URL = "jdbc:mysql://localhost:3306/tranquil_terrace";
    private static final String USER = "db_username";
    private static final String PASSWORD = "db_password";

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }
}

This snippet demonstrates how to use JDBC to connect to a MySQL database. The connection parameters are defined as constants, and the getConnection() method returns a live connection to the database.

Code Highlight: Reservation Management

Efficient reservation management is at the heart of the HMS. Below is a code snippet that sorts reservations alphabetically by room number and filters them based on customer name:

Java Code: Sorting and Filtering Reservations

List<Reservation> sortedReservations = reservations.stream()
    .sorted(Comparator.comparing(Reservation::getRoomNumber))
    .collect(Collectors.toList());

String searchQuery = "Deluxe";
List<Reservation> filteredReservations = sortedReservations.stream()
    .filter(r -> r.getCustomerName().toLowerCase().contains(searchQuery.toLowerCase()))
    .collect(Collectors.toList());

This code utilizes Java Streams to sort the list of reservations by room number and then filter it by matching the customer's name. This makes it easier for hotel staff to quickly access specific reservation data.

                // Method to capture employee face image using OpenCV
            private void captureEmployeeFace(String employeeId) {
                // Initialize the camera (device index 0 is usually the default webcam)
                VideoCapture camera = new VideoCapture(0);
                if (!camera.isOpened()) {
                    JOptionPane.showMessageDialog(this, "Camera not found! Please ensure a webcam is connected.");
                    return;
                }

                Mat frame = new Mat();
                // Attempt to read a frame from the camera
                if (camera.read(frame)) {
                    // Define the output file path (you could also store this in a database)
                    String filename = "employee_faces/" + employeeId + ".jpg";
                    // Write the captured frame to file in JPG format
                    Imgcodecs.imwrite(filename, frame);
                    JOptionPane.showMessageDialog(this, "Face captured successfully and saved as " + filename);
                } else {
                    JOptionPane.showMessageDialog(this, "Failed to capture face image. Please try again.");
                }
                
                // Release the camera resource
                camera.release();
            }

    Java Code: Exception handling and the use of a utility

                private void refreshHRData() {
                try {
                    Connection conn = DatabaseConnection.getConnection();
                    String sql = "SELECT * FROM hr_department";
                    PreparedStatement pst = conn.prepareStatement(sql);
                    ResultSet rs = pst.executeQuery();
                    // Assuming hrTable is your JTable and DbUtils is a helper library
                    hrTable.setModel(DbUtils.resultSetToTableModel(rs));
                } catch (SQLException e) {
                    JOptionPane.showMessageDialog(this, "Error refreshing HR data: " + e.getMessage());

            }

    Code Highlight: Dynamic Employee Search Functionality

    This snippet illustrates how to filter HR records based on a search query. The method uses a parameterized SQL query to search for employees whose names contain the search term (case-insensitive), then updates the table with the filtered results.

    Java Code: Parameterized SQL query to search for employees

                private void searchEmployee(String searchText) {
                try {
                    Connection conn = DatabaseConnection.getConnection();
                    String sql = "SELECT * FROM hr_department WHERE LOWER(employee_name) LIKE ?";
                    PreparedStatement pst = conn.prepareStatement(sql);
                    pst.setString(1, "%" + searchText.toLowerCase() + "%");
                    ResultSet rs = pst.executeQuery();
                    // Update the HR table with the search results
                    hrTable.setModel(DbUtils.resultSetToTableModel(rs));
                } catch (SQLException ex) {
                    JOptionPane.showMessageDialog(this, "Search error: " + ex.getMessage());
                    ex.printStackTrace();
                }
            }

    Challenges & Ongoing Improvements

    Concurrent Alert System for Periodic Notifications

    A major challenge was implementing a concurrent alert system that dispatches notifications every 15 minutes. Due to background processing limitations on mobile devices, the alert mechanism sometimes fails to trigger reliably. Although the AlarmManager-based approach works well in the foreground, background execution remains inconsistent. I’m currently investigating alternative strategies, such as integrating WorkManager, to improve reliability.

Key OOP Concepts

Object-Oriented Programming organizes software design around objects rather than actions, focusing on data and the methods that operate on that data. The four core pillars are:

  • Abstraction: Hiding complex implementation details and showing only essential features.
  • Inheritance: Establishing an "IS-A" relationship where a subclass inherits attributes and methods from a superclass.
  • Encapsulation: Bundling data and methods within classes to protect the internal state.
  • Polymorphism: Allowing entities to take on multiple forms through method overriding or overloading.

Valuable Code Snippets

1. Abstraction in Action

{`abstract class Vehicle {
    int wheels;
    public abstract void drive();
}

class Car extends Vehicle {
    @Override
    public void drive() {
        System.out.println("Car is driving smoothly.");
    }
}

// Usage
Vehicle myCar = new Car();
myCar.drive();`}

This code demonstrates abstraction by exposing only the essential driving behavior while hiding complex details.

2. Inheritance and IS-A Relationship

{`class Animal {
    public void makeSound() {
        System.out.println("Some generic sound");
    }
}

class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Bark!");
    }
}

// Usage
Animal pet = new Dog();
pet.makeSound();`}

This snippet illustrates inheritance: a Dog is an Animal, but with specialized behavior.

3. Encapsulation Example

{`class Employee {
    private String name;
    private double salary;
    
    public Employee(String name, double salary) {
        this.name = name;
        this.salary = salary;
    }
    
    public String getName() {
        return name;
    }
    
    public double getSalary() {
        return salary;
    }
    
    public void setSalary(double salary) {
        if(salary > 0) {
            this.salary = salary;
        }
    }
}

// Usage
Employee emp = new Employee("Tharindu", 50000);
System.out.println(emp.getName());
emp.setSalary(55000);`}

This example encapsulates employee data, ensuring that sensitive properties are accessed only through controlled methods.

4. Polymorphism Example

{`class Shape {
    public void draw() {
        System.out.println("Drawing a shape");
    }
}

class Circle extends Shape {
    @Override
    public void draw() {
        System.out.println("Drawing a circle");
    }
}

class Square extends Shape {
    @Override
    public void draw() {
        System.out.println("Drawing a square");
    }
}

// Usage
Shape s1 = new Circle();
Shape s2 = new Square();
s1.draw();
s2.draw();`}

This snippet shows polymorphism by allowing a Shape reference to point to different concrete shapes and call their respective draw methods.

5. Overloading and Overriding

{`class Calculator {
    // Method Overloading: same method name, different parameters
    public int add(int a, int b) {
        return a + b;
    }
    
    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

class AdvancedCalculator extends Calculator {
    // Method Overriding: providing a specific implementation for a method from the superclass
    @Override
    public int add(int a, int b) {
        System.out.println("Performing advanced addition:");
        return super.add(a, b);
    }
}

// Usage
Calculator calc = new AdvancedCalculator();
System.out.println(calc.add(10, 20));
System.out.println(calc.add(10, 20, 30));`}

This code highlights both method overloading and overriding, showcasing how a class can provide multiple implementations and specialized behavior.

Final Thoughts

Object-Oriented Programming (OOP) provides a powerful paradigm for developing complex software systems by promoting modularity, reusability, and maintainability. This post introduced key OOP concepts with real code examples, making it easier for beginners to grasp the fundamental principles of abstraction, inheritance, encapsulation, and polymorphism.

Whether you're a novice or an experienced developer, understanding these concepts will pave the way for writing cleaner, more efficient, and scalable code.
